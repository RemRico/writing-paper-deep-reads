## 章节裁剪指引

- **章节顺序固定**，但内容可裁剪：实验/消融如果论文做得少，对应小节可以缩短，但**不要删整节**
- 论文有"作者背景值得展开"（如阿里同事、新机构、有故事的关键人物）→ 在元信息后插入 `## 作者背景` 小节（参考 ResRank 那篇）
- 论文方法非常复杂、含多个 Stage → 4. 方法详解 可拆成 `4.1 Stage 1 / 4.2 Stage 2 / ...`，每个 Stage 内部再分"在干什么 / 数据 / 架构 / 损失"
- 论文有不熟悉的关键技术（如 Circle Loss、ColBERT 的 late interaction）→ 在该方法首次出现的子节加一个 `#### 📘 {技术名} 入门解读` 前置课
- 一句话总结 / TL;DR 不强制；但是非常长的论文（>20 页）建议加，方便日后扫读
